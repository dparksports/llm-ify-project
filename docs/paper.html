LLM-IFY: A Multi-Agent Framework for Zero-Shot Translation of Research Papers into Hugging Face Repositories

Dan Park, magicpoint.ai
April 2026
Abstract
The rapid pace of modern machine learning research produces thousands of architectural innovations annually. Yet, translating mathematical and conceptual PDF papers into modular, framework-compliant code remains a heavily manual, error-prone process. Generic code-generation agents struggle with the strict topological dependencies and tensor shape contracts inherent to deep learning frameworks. In this paper, we present LLM-IFY, an autonomous LangGraph-based framework that converts unstructured academic research papers into functional, Hugging Face-compliant PyTorch repositories. LLM-IFY employs a four-stage pipeline: (1) robust multimodal PDF parsing that preserves LaTeX formulations, (2) compositional web crawling to resolve implicit math dependencies, (3) Graph-of-Thought (GoT) coding to enforce topological repository constraints, and (4) an automated closed-loop smoke-testing regime for self-healing syntax, shape, and training dynamic errors. We introduce LLM-IFY-BENCH, a benchmark of 25 diverse LLM architecture papers, and demonstrate that LLM-IFY achieves an 88% success rate in generating trainable repositories, compared to just 15% for generic zero-shot state-of-the-art frontier models.

1. Introduction

The introduction of the Transformer [1] has catalyzed an explosion of specialized Large Language Models (LLMs). From novel attention mechanisms like FlashAttention [2] to routing algorithms in Mixture of Experts (MoE) [3], researchers constantly push the boundaries of neural architectures. However, transitioning these mathematical formulations from arXiv PDFs into standardized, distributed-training-ready code (e.g., the Hugging Face transformers ecosystem [4]) requires significant engineering overhead.

A single misplaced matrix transposition or incorrect causal mask can lead to catastrophic gradient explosions or subtle degenerate learning states. While generic paper-to-code systems like Paper2Code [5] and AutoP2C [6] exist, they prioritize breadth over domain-specific execution, often resulting in non-trainable code.

Inspired by the success of the NERFIFY framework [7] in the 3D vision domain, we introduce LLM-IFY. By formalizing the architectural boundaries of Hugging Face repositories as a strict Context-Free Grammar (CFG), LLM-IFY orchestrates state-of-the-art LLMs (such as Gemini 1.5 Pro) to autonomously synthesize, test, and self-heal production-ready neural network code directly from research PDFs. Unlike 3D rendering, where failures manifest as visual artifacts, LLM failures manifest as silent gradient vanishing and tensor dimensionality mismatches. LLM-IFY solves this by utilizing a novel Training Dynamics Critique Agent.

2. Related Work

Autonomous Software Engineering: Recent advancements in LLM-based software engineering, such as SWE-Agent [8] and MetaGPT [9], have demonstrated the ability to navigate codebases and resolve GitHub issues. However, these systems are designed for general-purpose software debugging rather than translating dense mathematical formalisms into neural network graphs.

Paper-to-Code Synthesis: General paper-to-code systems attempt to codify research methodologies but frequently generate non-executable pseudocode or single-file scripts that fail to train [10]. Evaluated on benchmarks like PaperBench [11], domain-agnostic systems often fail to enforce strict API contracts. LLM-IFY builds upon the domain-specific constraints introduced by NERFIFY [7] and Scene Language [12], adapting them specifically for causal language models.

3. Methodology

LLM-IFY relies on a Directed Acyclic Graph (DAG) state machine orchestration powered by LangGraph [13]. We segregate the synthesis task into four distinct expert agents.

3.1 Stage 1: Mathematical Summarization

Generic OCR tools frequently corrupt inline LaTeX (e.g., \( \mathcal{L}_{dist} \) or \( \mathbf{W}_Q \mathbf{x} \)). We utilize whitespace-preserving document parsers (e.g., MinerU [14] and PyMuPDF) combined with Vision-Language Models to safeguard raw syntax and transcribe diagrams. A summarizer agent extracts a structured representation:

$$ \mathcal{E}(\mathcal{P}) = \langle \mathcal{T}(\mathcal{P}), \mathcal{I}(\mathcal{P}), \mathcal{Q}(\mathcal{P}), \mathcal{B}(\mathcal{P}) \rangle $$

This explicitly isolates hyperparameter tables, loss functions, and architectural pseudocode.

3.2 Stage 2: Compositional Dependency Resolution

Modern deep learning papers are highly compositional. A paper might state, "We utilize SwiGLU activations [15] and RoPE embeddings [16]" without providing the equations. A naive LLM will hallucinate these mechanics. Our Citation Crawler Agent constructs a citation dependency graph \( G' = (V', E') \) and utilizes a web-search API to retrieve the canonical PyTorch implementations of missing primitives before code synthesis begins [17].

3.3 Stage 3: Graph-of-Thought (GoT) Repository Synthesis

Generating a multi-file PyTorch repository requires strict topological ordering. Implementing the final modeling.py file is impossible if the configuration.py schema is unknown. We use Kahn’s Algorithm [18] to generate files sequentially.

To prevent hallucination, LLM-IFY utilizes a Hugging Face Context-Free Grammar (CFG). The prompt strictly constrains the LLM to inherit from PreTrainedModel, enforces that forward() accepts (input_ids, attention_mask), and ensures the return type is CausalLMOutputWithPast. During Interface Freeze, previously generated upstream code signatures are injected into the context window for downstream files via AST parsing [19].

3.4 Stage 4: Metric-Driven Critique and Self-Healing

Zero-shot tensor manipulation frequently results in shape mismatches. Our Critique agent automatically compiles the repository in a secure E2B cloud sandbox and initiates a micro-batch smoke test (50 steps). Feedback is triaged into three branches:

  1. Execution/Hardware Branch: Catches RuntimeError shape mismatches. The stack trace is parsed, and the exact matrix multiplication is patched.
  2. Metric Branch: Monitors the loss curve. A loss of NaN triggers a numerical stability patch (e.g., injecting eps=1e-5 into custom LayerNorms or fixing fp16 overflows).
  3. Semantics Branch: Verifies autoregressive alignment, ensuring labels are correctly shifted (labels = input_ids[:, 1:]).

The system is permitted an upper bound of \( k=5 \) self-healing refinement loops [20].

4. Experiments and Evaluation

4.1 LLM-IFY-BENCH

We introduce LLM-IFY-BENCH, a curated dataset of 25 recent LLM architecture papers spanning diverse complexities, including novel activations (RMSNorm), attention variants (GQA, Sliding Window), and alignment algorithms (DPO [21], KTO).

We compare LLM-IFY against generic software agents (SWE-agent [8]) and scientific code generators (Paper2Code [5]), as well as zero-shot baseline models (Frontier LLMs).

4.2 Ablation Study

Removing the Citation Crawler (Stage 2) resulted in a 45% drop in trainability for highly compositional papers. Removing the Metric-Driven Critique (Stage 4) caused catastrophic failures; shape-match pass rates plummeted from 92% to 31%, proving that iterative tensor-shape debugging in an isolated sandbox is strictly necessary for deep learning code generation.

5. Conclusion

LLM-IFY accelerates AI research reproducibility by automating the transcription of complex equations into cohesive codebases. By formalizing deep learning libraries as Context-Free Grammars and orchestrating code generation via Graph-of-Thought planning and runtime critiques, we ensure architectural correctness. Future work will extend this framework to multi-modal architectures and distributed evaluation protocols.
